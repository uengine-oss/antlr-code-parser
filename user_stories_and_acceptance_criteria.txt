5. User Stories

User Story 1
As a 온라인 쇼핑 고객
I want 내가 주문한 상품의 현재 배송 상태를 한 화면에서 확인하고
So that 배송 진행 상황을 예측하고 불안함을 줄일 수 있다.

User Story 2
As a 고객센터 상담원
I want 고객 주문의 배송 상태를 즉시 확인하고
So that 고객 문의에 빠르게 대응할 수 있다.


7. 인수 조건 (Acceptance Criteria - Given / When / Then)

AC-1. 주문 상태 기본 표시

Given
- 사용자가 로그인된 상태이며
- 최소 1건 이상의 주문 이력이 존재한다

When
- 사용자가 주문 상세 페이지에 진입하면

Then
- 현재 주문 상태가 명확한 텍스트로 표시되어야 한다
- 상태는 사전에 정의된 값(주문 완료 / 상품 준비 중 / 배송 중 / 배송 완료) 중 하나여야 한다

AC-2. 상담원 주문 배송 상태 조회

Given
- 고객센터 상담원이 로그인된 상태이며
- 조회할 주문 번호가 존재한다

When
- 상담원이 주문 번호를 입력하여 배송 상태를 조회하면

Then
- 해당 주문의 배송 상태가 즉시 표시되어야 한다
- 상태는 사전에 정의된 값(주문 완료 / 상품 준비 중 / 배송 중 / 배송 완료) 중 하나여야 한다
- 조회 결과는 3초 이내에 표시되어야 한다
